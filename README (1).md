# 人才保留風險分析：用分群打破平均值的假象

**吳育庭**　|　工具：Python (pandas) · Power BI (DAX)
**資料來源**：[IBM HR Analytics Employee Attrition Dataset](https://www.kaggle.com/datasets/pavansubhasht/ibm-hr-analytics-attrition-dataset)（公開資料集，1,470 名員工、34 個變數）

## 想知道的問題

公司整體離職率是16.1%，乍看沒什麼問題，跟業界水準差不多。但這種公司層級的平均數字，很容易把局部嚴重的問題蓋掉——如果某個職位或某群人的離職率特別高,但人數佔比不大,平均值看起來還是會很「正常」。

這份分析想找的是：平均值底下藏了哪些被忽略的高風險族群？原因是什麼？可以先從哪裡下手?

## 怎麼做的

先一個一個變數看離職率的差異，確認方向後再看複合風險（同時中兩個以上風險因子的人），最後把幾個因子組成一個分數，方便排序。

完整過程在 [`talent_retention_analysis.ipynb`](./talent_retention_analysis.ipynb)。

## 看到了什麼

**Sales Representative 的離職率是 39.8%，全公司平均才16.1%**
這個職位的人數不算多（n=83），所以放進公司整體數字裡會被稀釋掉，但拆開來看，差距快3倍，不是隨機誤差。相對地，Manager、Research Director這些資深職位離職率只有2.5%–4.9%。

**加班是影響最大的單一因素**
有加班的人離職率30.5%，沒加班的只有10.4%。這個如果要處理，成本相對低——看排班、看人力配置就能動手。

**加班疊上升遷停滯，風險會放大**
同時加班又超過3年沒升遷的人，離職率28%；相對地，沒加班又近期有升遷的人只有11.1%。這種複合風險的人，分散在不同部門裡，個別部門報表通常看不出來。

## 風險分數

把幾個因子依照各自造成的離職率差異幅度設權重，組合成一個分數，方便排序「誰該先處理」：

```
風險分數 = 加班(0.35) + 升遷停滯3年以上(0.25) + 工作滿意度低(0.25) + 薪資低於中位數(0.15)
```

分數分高/中/低三層後，實際離職率確實隨分數升高，方向是對的。這個邏輯之後會用DAX在Power BI重做一次，做成互動儀表板。

## 大概值多少錢

如果Sales Representative的離職率能從39.8%降到接近全公司平均的16%,估算每年大概能少流失20人。用業界常見的離職成本估算法（招募+訓練+生產力空窗，約年薪的0.5–1.5倍）換算，一年大概可以省下31萬到93萬（用這份資料集的薪資單位算的示意性數字，真實案子要用實際薪資結構重新校準）。

## 這份分析還不夠嚴謹的地方

- 這是橫斷面資料，只能看相關性、排風險高低，不能說「加班導致離職」
- 權重是憑單變數差異幅度設的，比較直覺，嚴謹一點應該用邏輯回歸重新驗證
- 真要落地的話，建議先開始做結構化離職面談，累積資料才能驗證行動方案有沒有用
